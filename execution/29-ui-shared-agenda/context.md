### Agenda Share Flow (api-prd.md)
- `POST /agendas/{id}/share` — Generate shareable URLs:
  - Client-facing (read-only, no auth) — this is the URL that leads to `/shared/{token}`
  - Internal (edit-enabled, auth required)
- Agendas can only be shared if `status = finalized`.

### DNS Routing (infra-prd.md)
- `api.domain.com/shared/*` -> API container (public, no auth)
- Note: The UI renders the page, but fetches the agenda data from the API's public `/shared/{token}` endpoint.

## Scope

### In Scope
- Shared agenda page at route `/shared/{token}` using PublicLayout
- No authentication required — public access via token in URL
- Server-side rendered for SEO and fast initial load
- iExcel branding — clean, professional branded header
- Formatted display of finalized Running Notes content:
  - Completed Tasks section
  - Incomplete Tasks section
  - Relevant Deliverables section
  - Recommendations section
  - New Ideas section
  - Next Steps section
- Print option (browser print dialog)
- PDF download option (generate PDF from rendered content)
- Graceful error handling for:
  - Invalid token (agenda not found)
  - Expired token (if token expiry is implemented)
  - Agenda not yet finalized (should not be reachable, but handle gracefully)
- Responsive layout for viewing on various devices

### Out of Scope
- Authentication (this page bypasses OIDC entirely)
- Editing controls (read-only)
- Internal comments (not visible in shared view)
- Version history (not visible in shared view)
- Sidebar navigation (PublicLayout has no sidebar)
- Client commenting or reactions on the shared agenda (open question in PRD — currently read-only only)

## Key Decisions
- This page uses **PublicLayout**, not DashboardLayout. No sidebar, no navigation, no authentication UI elements.
- The page is **server-side rendered** for fast load and because no client-side JavaScript is needed for interactivity (read-only content).
- The token in the URL is the `shared_url_token` generated by `POST /agendas/{id}/share`. It is scoped to a specific agenda and client.
- Internal comments from the agenda editor are **never** displayed on this page. The shared view shows only the finalized Running Notes content.
- The PDF export can be implemented via browser's built-in print-to-PDF or a server-side PDF generation library. The design inspiration shows a "Download the Agenda as PDF" button.
- The open question about whether clients can leave comments or reactions is resolved as **read-only only** for V1. The PRD lists this as an open question.
