## Phase 4: Service Layer

### Task 4.1 тАФ Implement WorkflowService.triggerIntake
**Size:** Medium
**Reference:** TR.md ┬з 5.1, FRS.md ┬з 2

In `apps/api/src/services/workflow.service.ts`:

- [ ] Define `WorkflowService` class with constructor accepting `WorkflowRepository`, `TranscriptRepository`, `ReconciliationService`, `MastraAdapter`, `AuditService`, `AppConfig`
- [ ] Implement `triggerIntake(callerId, clientId, transcriptId)`:
  - Fetch and verify transcript belongs to client (throw `UnprocessableError('TRANSCRIPT_NOT_FOUND')` if not)
  - Check for active run (throw `ConflictError('WORKFLOW_ALREADY_RUNNING')` if found)
  - Create workflow run record with status 'pending'
  - Write audit log entry `workflow.triggered`
  - Fire-and-forget: call `mastraAdapter.invokeWorkflowA(...)`, catch errors with `handleInvocationFailure`
  - Return the created run record

**Verification:** Unit tests cover all precondition failure paths; success path; verify Mastra adapter is called exactly once; verify audit log is written; verify invocation failure marks run as failed without throwing.

### Task 4.2 тАФ Implement WorkflowService.triggerAgenda
**Size:** Medium
**Reference:** TR.md ┬з 5.1, FRS.md ┬з 3

In `apps/api/src/services/workflow.service.ts`:

- [ ] Implement `triggerAgenda(callerId, clientId, cycleStart?, cycleEnd?)`:
  - Check for active run
  - Resolve cycle dates: use provided values; else call `resolveCycleStart(clientId)` for start and `new Date()` for end
  - Call `reconciliationService.reconcileClient(clientId)`
  - Count completed tasks in date range; throw `UnprocessableError('NO_COMPLETED_TASKS')` with human-readable message if zero
  - Create workflow run record with status 'pending' and input_refs containing resolved dates
  - Write audit log entry `workflow.triggered`
  - Fire-and-forget Mastra Workflow B invocation
  - Return run record
- [ ] Implement `resolveCycleStart(clientId)` тАФ fetch last completed agenda run; fall back to 30 days ago

**Verification:** Unit tests cover: no completed tasks returns 422; active run returns 409; success path creates record and fires Mastra; cycle date resolution uses last run date; 30-day fallback when no prior run.

### Task 4.3 тАФ Implement WorkflowService.getStatus
**Size:** Small
**Reference:** TR.md ┬з 5.1, FRS.md ┬з 7

- [ ] Implement `getStatus(callerId, workflowRunId)`:
  - Fetch run by ID (repository throws 404 if not found)
  - If status is 'pending' or 'running', compute age from `updated_at`
  - If age exceeds `WORKFLOW_TIMEOUT_MS`, call `markTimedOut(run)` and return the updated run
  - Otherwise return the run as-is
- [ ] Implement `markTimedOut(run)` private method:
  - Update status to 'failed', set error `{ code: 'WORKFLOW_TIMEOUT', message: '...' }`, set completed_at
  - Write audit log entry `workflow.timed_out`
  - Return updated run

**Verification:** Unit tests: pending run within timeout is returned unchanged; run past timeout is marked failed; completed run is returned unchanged; failed run is returned unchanged.
