# Glossary

Domain-specific terms used throughout the iExcel Automation codebase.

## A

**Agenda** - A meeting preparation document generated from reconciled tasks. Contains structured ProseMirror JSON content with sections for each task's status, next steps, and discussion points. Delivered via Google Docs and Email adapters.

**Agenda Agent** - Mastra AI agent (`agents/agenda-handler.ts`) that processes reconciled task data to generate structured meeting agendas. Outputs ProseMirror JSON content.

**Asana Client** - Shared HTTP client (`adapters/asana/asana-client.ts`) for Asana API operations. Handles authentication, rate limiting, and error handling. Used by output normalizer, reconciliation, and historical import.

## B

**BullMQ** - Redis-backed job queue used by Mastra for background workflow processing. Workers consume jobs for intake and agenda agent execution.

## C

**Client** - An organizational entity (company/team) whose meetings are tracked. Has associated users (via `client_users` join table), transcripts, tasks, and agendas. Identified by UUID and short ID.

**Client Credentials Flow** - OAuth2 grant type used for service-to-service authentication (API ↔ Mastra). Issues service tokens without user context.

## D

**Device Flow** - OAuth2 device authorization grant used for terminal/CLI authentication. User approves login in browser; terminal polls for token. Implemented in `packages/terminal-auth/`.

**Drizzle ORM** - TypeScript ORM used for all database operations and migrations. Schema definitions in `packages/database/`.

## E

**EmailRecipient** - Type defining agenda email recipients: `{ name, email, role? }`. Stored as JSONB array on the `clients` table.

**External Ref** - JSONB field on tasks (`external_ref`) linking to external PM tools. Standardized shape: `{ system, externalId, externalUrl, projectId, workspaceId }`. Currently only Asana; architecture supports multi-PM-tool expansion.

## G

**Grain** - Meeting recording platform. V1 supports manual transcript paste/upload. V2 (Feature 37) adds direct Grain API integration via `normalizers/grain/grain-client.ts`.

## H

**Historical Import** - Feature 38: bulk import of existing Asana tasks into iExcel. Uses dedicated `import_jobs` and `import_job_errors` tables with own Drizzle migration.

## I

**Import Job** - Tracks a historical import operation from Asana. States: `pending → processing → completed/failed`. Errors logged to `import_job_errors` table.

**Input Normalizer** - Layer 1 of the two-layer normalization pattern. Converts source-specific transcript formats (text, Grain) into canonical `NormalizedTranscript` format. Located in `apps/api/src/normalizers/`.

**Intake Agent** - Mastra AI agent (`agents/intake-agent.ts`) that processes `NormalizedTranscript` to extract structured `NormalizedTask[]` with context, assignees, priorities, and time estimates.

## M

**Mastra** - AI agent orchestration framework used for running intake and agenda agents. Provides tool definitions, workflow execution, and MCP server capabilities.

**MCP (Model Context Protocol)** - Protocol enabling Claude Code terminal access to iExcel tools. Mastra exposes 10 MCP tools with user token passthrough for authenticated operations.

## N

**NormalizedTask** - Canonical task type defined in `@iexcel/shared-types`. Contains: `title`, `description` (TaskDescription JSONB), `priority`, `estimatedTime` (ISO 8601), `assignee`, `externalRef`.

**NormalizedTranscript** - Canonical transcript type after input normalization. Contains `normalizedSegments` (JSONB) with speaker-attributed, timestamped content.

## O

**Output Normalizer** - Layer 2 of the two-layer normalization pattern. Converts internal `NormalizedTask` to external PM tool format. Currently: Asana adapter. Located in `apps/api/src/services/output-normalizer.ts`.

## P

**ProseMirror JSON** - Structured content format for agenda storage. Native to TipTap editor (UI Feature 28). Google Docs adapter parses ProseMirror nodes for document generation.

## R

**Reconciliation** - Bidirectional status sync between internal tasks and Asana. Queries Asana for task completion status, updates internal DB cache. Implemented in `adapters/asana/reconcile.ts`.

## S

**Short ID** - Human-readable identifier using 3+ digit format (e.g., `TSK-001`, `AGD-042`, `CLT-007`). Regex: `\d{3,}` (uncapped). Resolved to UUIDs via middleware.

**Status Reconciliation** - See **Reconciliation**.

## T

**TaskDescription** - Structured JSONB type: `{ taskContext, additionalContext, requirements }`. Stored as JSONB in Postgres `tasks.description` column. Generated by intake agent.

**TaskStatus** - Enum: `draft | approved | rejected | pushed | completed`. Normal flow: `draft → approved → pushed`. `completed` set only by reconciliation or historical import.

**TaskPriority** - Enum: `low | medium | high | critical`. Enforced by CHECK constraint in Postgres.

**Token Manager** - Terminal authentication component (`packages/terminal-auth/src/auth/token-manager.ts`) that caches and refreshes JWT tokens for CLI sessions.

## W

**Workflow Orchestration** - Feature 17: coordinates the execution of intake and agenda workflows. Triggers Mastra agents via API, tracks execution in `workflow_runs` table.

**Workflow Run** - Record tracking a single workflow execution. States: `pending → running → completed/failed`. Owned by Feature 17 with its own Drizzle migration.
